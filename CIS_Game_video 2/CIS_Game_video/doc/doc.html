积分墙推荐墙使用文档

推荐墙相当于无积分的积分墙，下面如无特别说明，适用积分墙的同时也适用于推荐墙。

积分墙各版本使用改动

4.8

添加 YouMiPointsManager

5.0

全新的使用方式

新版积分墙有重大更改，SDK使用方式与以往有所不同，请重新阅读教程

嵌入步骤

1. 注册应用，申请ID

请登陆到 www.youmi.net/sdk 注册一个开发者账号，同时选择注册一个应用，获取该应用的 appid 和 appsecret 。

2. 下载YouMi SDK iOS 压缩包

3. 往工程导入头文件， 静态库文件

积分墙必须的头文件为
  • YouMiConfig.h
  • YouMiWall.h
  • YouMiWallAppModel.h
  • YouMiPointsManager.h (用于查询积分)
静态库文件为
  • libYouMi.a

4. 为工程添加Frameworks

  • SystemConfiguration.framework
  • CFNetwork.framework
  • QuartzCore.framework
  • libsqlite3.dylib

使用方法

1. 在AppDelegate.m中初始化【必须】{: #section_wall_init .sdkhead}

  • 导入 YouMiWall.h
#import "YouMiWall.h"
  • 在 application:didFinishLaunchingWithOptions: 中设置SDK共用属性(可选)(必须在设置appid, appsecret之前) appid 和 appsecret
[YouMiConfig setUserID:id_you_define]; // 可选 (用于帮助开发者区分用户，例如应用有不同账号登陆，可以把账号名设置为userID，如果不设置，则返回积分时userID字段为"")
[YouMiConfig setUseInAppStore:YES];  // 可选，详细请看YouMiSDK常见问题解答
[YouMiConfig launchWithAppID:@"[Your AppID]" appSecret:@"[Your AppSecret]"];

  • 设置显示全屏广告（如积分墙）的全屏UIWindow(可选，推荐设置)

    对于使用UIKit编写的APP可以在application:didFinishLaunchingWithOptions:中的[self.window makeKeyAndVisible]之后设置：

[self.window makeKeyAndVisible];
// 设置显示全屏广告的window   
[YouMiConfig setFullScreenWindow:self.window];

2. 启用积分墙【必须】{: #section_wall_enable .sdkhead}

告诉有米SDK使用到的广告形式是积分墙. 在设置 appid appsecret之后调用:

[YouMiWall enable];

3. 启用积分管理

  • 【使用有积分广告时若使用SDK客户端回调，必须】

让有米SDK从服务器查询积分，并通知；对于 自动积分管理 以及 手动积分管理 的详细解释，请看后面 5. 获取积分

在设置 appid appsecret之后调用:

若使用自动积分管理

[YouMiPointsManager enable];

若使用手动积分管理

[YouMiPointsManager enableManually];

4. 显示

显示积分墙：

[YouMiWall showOffers:YES didShowBlock:^{
    NSLog(@"有米积分墙已显示");
} didDismissBlock:^{
    NSLog(@"有米积分墙已退出");
}];

显示推荐墙：

[YouMiWall showOffers:NO didShowBlock:^{
    NSLog(@"有米推荐墙已显示");
} didDismissBlock:^{
    NSLog(@"有米推荐墙已退出");
}];

5. 获取积分

我们提供了 "SDK客户端回调" 和 "服务器回调" 两种获取积分方式。

  • YouMiPointsManager (YouMiSDK4.8之后提供):
  • 服务器回调: 用户完成安装(或者注册等步骤)后有米服务器向开发者的服务器发送积分数据，由开发者服务器负责管理积分
  • 客户端开发者调用的查询接口在 YouMiSDK 5.0 以及之后取消。
YouMiPointsManager

从YouMiSDK 5.0开始 YouMiPointsManager 有两种使用方式：

  1. 自动积分管理(同YouMiSDK 4.8)

    YouMiSDK 从服务器接收到积分后，发出通知的同时，会把积分存在客户端上，供开发者调用spendPoints方法消耗；

    自动积分管理简单方便；

    对于一个有多个账户的APP，自动积分管理无法区分哪些积分属于哪个账户。

  2. 手动积分管理

    YouMiSDK 从服务器接收到积分后，发出的通知，里面包含了各份积分属于哪个账户(userID)，开发者接收到通知后自己处理拿到的积分。

    手动积分管理无法使用spendPoints，pointsRemained方法。

使用YouMiPointsManager，首先需要启动YouMiPointsManager 启动方法如前面 3. 启用积分管理

自动积分管理使用方法如下：

查询剩余积分

[YouMiPointsManager pointsRemained];

消耗剩余的积分

[YouMiPointsManager spendPoints:10];

当用户获得新的积分的时候，YouMiSDK(不是YouMiPointsManager)会发出一个 kYouMiPointsManagerRecivedPointsNotification。

[[NSNotificationCenter defaultCenter] addObserver:self selector:@selector(pointsGotted:) name:kYouMiPointsManagerRecivedPointsNotification object:nil];

- (void)pointsGotted:(NSNotification *)notification {
    NSDictionary *dict = [notification userInfo];
    NSNumber *freshPoints = [dict objectForKey:kYouMiPointsManagerFreshPointsKey];

    // 这里的积分不应该拿来使用, 只是用于告知一下用户, 可以通过 [YouMiPointsManager spendPoints:]来使用积分
    UIAlertView *alert = [[UIAlertView alloc] initWithTitle:@"通知" message:[NSString stringWithFormat:@"获得%@积分", freshPoints] delegate:nil cancelButtonTitle:@"好的" otherButtonTitles:nil];
    [alert show];
    [alert release];

    self.pointsLabel.text = [NSString stringWithFormat:@"当前积分: %d", [YouMiPointsManager pointsRemained]];
}

手动积分管理使用方法如下：

当用户获得新的积分的时候，YouMiSDK(注意可能不是YouMiPointsManager)会发出一个 kYouMiPointsManagerRecivedPointsNotification。

[[NSNotificationCenter defaultCenter] addObserver:self selector:@selector(pointsGotted:) name:kYouMiPointsManagerRecivedPointsNotification object:nil];

- (void)pointsGotted:(NSNotification *)notification {
    NSDictionary *dict = [notification userInfo];
    // 手动积分管理可以通过下面这种方法获得每份积分的信息。
    NSArray *pointInfos = dict[kYouMiPointsManagerPointInfosKey];
    for (NSDictionary *aPointInfo in pointInfos) {
        // aPointInfo 是每份积分的信息，包括积分数，userID，下载的APP的名字
        NSLog(@"积分数：%@", aPointInfo[kYouMiPointsManagerPointAmountKey]);
        NSLog(@"userID：%@", aPointInfo[kYouMiPointsManagerPointUserIDKey]);
        NSLog(@"产品名字：%@", aPointInfo[kYouMiPointsManagerPointProductNameKey]);

        // TODO 按需要处理
    }
}

6. 使用源数据

通过YouMiSDK能够方便获取到APP广告列表，我们为开发者提供APP的实用信息，方便开发者自定义界面。

获取APP的方法如下（以有积分模式为例）：

[YouMiWall requestOffersOpenData:YES revievedBlock:^(NSArray *theApps, NSError *error) {
            // theApps 是 YouMiWallAppModel 的对象列表
        if (!error) {
            self.apps = theApps;
            [self.table reloadData];
        }
}];

当用户点击一个APP时，调用userInstallApp:方法安装APP：

- (void)tableView:(UITableView *)tableView didSelectRowAtIndexPath:(NSIndexPath *)indexPath {
    [tableView deselectRowAtIndexPath:indexPath animated:YES];
    YouMiWallAppModel *model = self.apps[indexPath.row];
    [YouMiWall userInstallApp:model];
}

更详细用法请看例子程序。

YouMiSDK 常见问题解答

1. YouMiSDK4.7以及之前的积分查询有什么需要注意的

  1. 同一个应用多次安装, 只会返回一次积分；
  2. 有米服务器不会保存被查询过的积分，需要开发者自行管理；
  3. 建议程序从后台进入前台是查询2-3次积分；
  4. 建议更新到YouMiSDK最新版本使用YouMiPointsManager来获得积分。

2. 是否使用了UDID

YouMiSDK 4.7 以及之后版本 不再使用UDID

3. 什么是内置AppStore

内置AppStore 是在应用内显示的AppStore页面。 如图 iPad 中的效果：

开启了内置AppStore，不需要跳出应用也能够安装App，但有若干限制

  1. iOS 6 或以上系统才有此功能。
  2. 没有通过AppStore审核的应用的 内置AppStore 只能够显示，不能下载。

4. iOS6系统内置AppStore无法下载安装，应该如何使用

YouMiSDK 提供 [YouMiConfig setUseInAppStore:] 控制开启关闭 内置AppStore。开发期间可以暂时关闭 内置AppStore, 建议提交审核前开启, 方便用户使用。

5. 编译时提示 Undefined symbols

这种情况一般发生在升级YouMiSDK时，或者和聚合SDK一起使用时。要作如下检查
1. 检查工程里有没有两份YouMiSDK (libYouMi.a 以及 头文件)， 如果有，请去掉旧的；
2. 检查工程目录下有没有两分YouMiSDK，如果有，请去掉旧的； 3. 检查编译Target里的Library Seach Paths 是否有两个YouMiSDK的路径，如果有请去掉旧的。

6. 如何取消获取用户地理位置

在 

[YouMiConfig launchWithAppID:@"Your AppID" appSecret:@"Your AppSecret"]

前使用

[YouMiConfig setShouldGetLocation:NO];

7. Banner广告重复点击是否有效

若一个广告被多次点击，只会记录第一次点击效果。重复的点击无效。

8. 有积分模式和无积分模式有什么区别

两者作用不同：有积分模式用于激励下载，用户通过下载安装APP来获得积分；无积分模式用于推荐。

9. 使用了YouMiView并设置了delegate，退出ViewController后偶尔崩溃

请记得在ViewController销毁的时候把 YouMiView 对象的 delegate 设置为nil，只要这样就不会崩溃。
